# Experiment Pipeline

Internal tooling for taste-labeling records and validating `RecordScorer` against
human-labeled ground truth. Each crate run produces a permanent holdout fixture so
every algorithm change is measured over time.

## Quick Start

End-to-end pipeline run:

```bash
# 1. Ensure your store is synced (seed generator needs available listings)
rake milkcrate:sync

# 2. Generate a stratified seed crate (~40 records across hot/warm/cold/lukewarm bands)
rake experiment:generate[crate-001]

# 3. Copy the labeling page into the crate directory
cp experiments/crate-001/label.html experiments/crate-001/  # already there after generate

# 4. Open the labeling page — Round 1 (blinded: cover + artist + title only)
open experiments/crate-001/label.html?round=1

# 5. Label all 40 records (~15 min), then click "Download Results"
#    Save the file as round1-results.json inside experiments/crate-001/

# 6. Next day: Round 2 (full metadata)
open experiments/crate-001/label.html?round=2

# 7. Re-label all 40 records (~10 min), download as round2-results.json
#    If your label changes from Round 1, a "What changed your mind?" field appears

# 8. Merge seed + labels into analysis CSV
ruby analysis/merge.rb experiments/crate-001/

# 9. Run analysis scripts
ruby analysis/seams.rb experiments/crate-001/results.csv
ruby analysis/logistic_regression.rb experiments/crate-001/results.csv
ruby analysis/anti_scorer.rb experiments/crate-001/results.csv
ruby analysis/absence_profile.rb experiments/crate-001/results.csv

# 10. Ablation requires Rails (run from project root)
ruby -I app analysis/ablation.rb experiments/crate-001/results.csv

# 11. Register the baseline score
ruby analysis/evaluate.rb experiments/crate-001/results.csv baseline

# 12. Populate the holdout fixture
# Copy labeled records into spec/fixtures/crate-001-labels.yml
```

## Directory Structure

```
experiments/                        ← per-session artifacts (gitignored)
├── score-registry.yml              ← append-only evaluation log (committed)
└── crate-001/                      ← one directory per experiment run
    ├── seed.json                   ← generated by experiment:generate
    ├── label.html                  ← labeling page (standalone HTML)
    ├── round1-results.json         ← downloaded after Round 1
    ├── round2-results.json         ← downloaded after Round 2
    └── results.csv                 ← produced by merge.rb

analysis/                           ← analysis scripts (committed)
├── README.md                       ← you are here
├── merge.rb
├── logistic_regression.rb
├── ablation.rb
├── anti_scorer.rb
├── seams.rb
├── absence_profile.rb
└── evaluate.rb

spec/fixtures/
└── crate-001-labels.yml            ← holdout fixture (populated after first run)
```

## Labeling Page

The labeling page (`label.html`) is a standalone HTML file — no server needed.
Open it directly from `file://` or serve with:

```bash
python3 -m http.server -d experiments/crate-001/
```

**Round 1** (`?round=1` — default): Cover image, artist, title. Buttons disabled
for 2 seconds (prevents reflexive clicking). Keyboard shortcuts: **J** = Junk,
**I** = Indifferent, **K** = Cool. No back button — first impressions only.

**Round 2** (`?round=2`): Same layout plus a metadata panel (year, genres, styles,
condition, price, want/have, label, format, band, algorithm score). If your label
differs from Round 1, an optional "What changed your mind?" text field appears.

Results auto-save to `localStorage`. The "Download Results" button exports JSON.

## Running Another Crate

```bash
rake experiment:generate[crate-002]
cp experiments/crate-001/label.html experiments/crate-002/
# ... label, merge, analyze ...
```

Each crate is independent. The score registry accumulates results across crates,
so you can measure whether a `RecordScorer` change actually improved agreement.

## Score Registry

`experiments/score-registry.yml` is an append-only log of evaluations. Each entry records:

- Date, crate name, version label
- Accuracy, precision, recall, F1
- Confusion matrix (TP/TN/FP/FN)
- Δ from previous entry (when available)

Never edit or delete previous entries — the registry is the permanent record.

## Holdout Fixture

After your first full pipeline run, populate `spec/fixtures/crate-001-labels.yml`
with the labeled records. This becomes a permanent test fixture that validates
future `RecordScorer` changes don't regress on known labels.

## Analysis Scripts

Plain Ruby scripts that read a merged `results.csv` and output findings to stdout.

### Script Reference

| Script | What it tells you | Requires Rails? |
|--------|-------------------|----------------|
| `merge.rb` | Merge seed JSON + labeling results → `results.csv` | No |
| `seams.rb` | Confusion matrix + which records the scorer gets wrong | No |
| `logistic_regression.rb` | Which strategy scores actually predict "cool" labels | No |
| `anti_scorer.rb` | Same regression inverted — what predicts "junk" | No |
| `absence_profile.rb` | Does missing metadata correlate with junk labels? | No |
| `ablation.rb` | What happens to agreement when you remove each strategy? | Yes |
| `evaluate.rb` | Accuracy/precision/recall/F1 + append to score registry | No |

### Usage

All scripts accept a path to `results.csv` as their first argument:

```bash
ruby analysis/seams.rb experiments/crate-001/results.csv
```

Scripts that need Rails model access (`ablation.rb`) must be run with the app
load path from the project root:

```bash
ruby -I app analysis/ablation.rb experiments/crate-001/results.csv
```

### Output Format

All scripts print to stdout with `=== Section Headers ===` separating logical
sections. Redirect to files for persistence:

```bash
ruby analysis/logistic_regression.rb experiments/crate-001/results.csv > experiments/crate-001/logistic-regression.txt
```
