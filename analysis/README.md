# Experiment Pipeline

Label the algorithm's top-scored records as junk, indifferent, or cool to find
where the scorer is surfacing records you disagree with.

## Quick Start

```bash
# 1. Generate a crate of top-scored records
bin/rake 'experiment:generate[crate-001]'

# 2. Start the labeling page
bin/rake 'experiment:serve[crate-001]'
#    → Open http://localhost:8000/label.html

# 3. Label all records, click "Download Results"
#    Save the file as results.json inside experiments/crate-001/

# 4. Run the report
bin/rake 'experiment:report[crate-001]'
```

## Directory Structure

```
experiments/                        ← per-session artifacts (gitignored)
└── crate-001/                      ← one directory per experiment run
    ├── seed.json                   ← generated by experiment:generate
    ├── label.html                  ← labeling page (standalone HTML)
    └── results.json                ← downloaded after labeling

analysis/
├── README.md                       ← you are here
├── label.html                      ← labeling page template
└── report.rb                       ← generates the experiment report
```

## Labeling Page

Standalone HTML — no server needed (the `experiment:serve` task runs
`python3 -m http.server` in the crate directory).

- Cover image, artist, title, and all metadata shown for each record
- Buttons disabled for 2 seconds (prevents reflexive clicking)
- Keyboard shortcuts: **J** = Junk, **I** = Indifferent, **K** = Cool
- Results auto-save to `localStorage`
- Click "Download Results" to export as `results.json`

## The Report

Run `bin/rake 'experiment:report[crate-001]'` after labeling. It prints:

1. **Score ranges per label** — mean and min/max scores for junk, indifferent,
   and cool. The mean scores should descend cool → indifferent → junk. If they
   don't, the algorithm isn't separating well.

2. **Not-cool records, highest-scoring first** — for each record that you
   labeled junk or indifferent, shows the per-strategy score breakdown so you
   can see *exactly why* the algorithm thought it was interesting:

   ```
   1. Artist Name — Album Title
      Label: indifferent  |  Score: +4.83
      2019  |  Hip Hop  |  Mint (M)  |  $19.95
      Want/Have: 47/115
      Score breakdown:
        vintage               -0.12
        condition              +1.50
        section                +0.80
        ...
   ```

3. **Cool records** — summary list with scores.

## Running Another Crate

```bash
bin/rake 'experiment:generate[crate-002]'
bin/rake 'experiment:serve[crate-002]'
# ... label, save results.json ...
bin/rake 'experiment:report[crate-002]'
```

Each crate is independent.
